# 0005-use-just-as-command-runner

- Status: accepted
- Date: 2026-02-16

## 选了什么
- 命令行任务编排器统一使用 `just`。

## 为什么
- `just` 语法简单，适合统一常用开发命令入口。
- 可降低跨脚本、跨命令的记忆成本，便于多管道协作保持一致。
- 与项目当前“先收敛流程，再扩展能力”的节奏一致。

## 放弃了什么
- 方案 A：继续使用散落的 shell 命令和文档约定。
  - 放弃原因：一致性差，团队协作与后续自动化成本更高。
- 方案 B：使用 Makefile 作为主任务入口。
  - 放弃原因：可读性和易用性不如 `just`，且对非 GNU 环境细节更敏感。

## 影响
- 运行、测试、格式化、检查等常用任务将逐步收敛到 `just` 命令。
- 后续新增开发流程默认先提供对应 `just` 任务入口。
