# 0004-use-uv-as-python-package-manager

- Status: accepted
- Date: 2026-02-16

## 选了什么
- Python 包管理器统一使用 `uv`。

## 为什么
- `uv` 在安装和解析依赖上速度快，适合快速迭代开发。
- 工具链简洁，可同时覆盖虚拟环境和依赖管理的核心场景。
- 与当前“精简实现、降低流程负担”的项目原则一致。

## 放弃了什么
- 方案 A：继续使用 Poetry。
  - 放弃原因：当前阶段更关注轻量和速度，工具复杂度可进一步降低。
- 方案 B：使用 pip + requirements.txt 手工管理。
  - 放弃原因：依赖锁定和工作流一致性较弱，长期维护成本更高。

## 影响
- 后续文档与开发指令中，包管理相关命令默认使用 `uv`。
- 新增或更新依赖时，以 `uv` 工作流为唯一基线。
