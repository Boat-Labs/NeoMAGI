# 0002-default-openai-with-gemini-migration-validation

- Status: accepted
- Date: 2026-02-16

## 选了什么
- NeoMAGI 在当前阶段默认采用 OpenAI API 作为主运行模型路线。
- 同时保留 Gemini API 作为迁移验证目标，纳入阶段性可验证范围。

## 为什么
- 先收敛到单一默认路线，有助于降低 M1-M4 阶段复杂度，优先验证核心用户价值。
- 保留 Gemini 迁移验证可避免早期平台锁定，降低中长期供应商依赖风险。
- 该策略与当前里程碑一致：先可用，再可迁移，不提前扩展到多模型运营复杂度。

## 放弃了什么
- 方案 A：从一开始双主路线并行（OpenAI 与 Gemini 同等优先级）。
  - 放弃原因：增加测试矩阵和决策复杂度，不利于当前阶段快速收敛。
- 方案 B：仅绑定 OpenAI，不做迁移验证。
  - 放弃原因：中长期替换成本不可控，缺乏迁移可行性证据。

## 影响
- 后续里程碑中的模型相关验收以“OpenAI 默认可稳定运行”为基线。
- 在 M6 阶段必须形成 Gemini 迁移验证结论，并给出可执行切换/回退判断。
