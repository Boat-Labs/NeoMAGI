# 0011-frontend-package-manager-pnpm-with-just-entrypoints

- Status: accepted
- Date: 2026-02-16

## 选了什么
- 前端包管理器采用 `pnpm`。
- 任务入口统一通过 `just` 暴露（对齐全仓库命令约定）。

## 为什么
- `pnpm` 依赖安装快、磁盘利用率高，适合前端依赖场景。
- 通过 `just` 暴露统一入口，可降低多管道协作下命令分散问题。
- 与现有决策（仓库级 `just` 命令入口）保持一致。

## 放弃了什么
- 方案 A：前端使用 npm/yarn，命令直接写在文档中手工执行。
  - 放弃原因：协作时命令口径容易分叉，缺乏统一入口。
- 方案 B：前端也使用 `uv` 管理 JS 依赖。
  - 放弃原因：不符合 JS 生态工具链常规，维护与生态兼容性差。

## 影响
- 前端依赖安装与升级默认走 `pnpm`。
- 对开发者暴露的常用前端命令由 `just` 统一编排与调用。
