# 0039-openai-primary-development-test-model-gpt-5-mini

- Status: accepted
- Date: 2026-02-24

## 选了什么
- 将开发测试（含本地联调、功能回归、里程碑验收中的主验证路径）默认 OpenAI 模型固定为 `gpt-5-mini`。
- 明确成本控制策略：
  - 默认先用 `gpt-5-mini` 跑主测试集；
  - 需要极低成本烟雾测试时，可临时降级为更低价模型（如 `gpt-5-nano`），但不作为主结论依据。
- 本决议仅约束“开发测试主模型”，不改变生产默认路由与供应商迁移决议。

## 为什么
- `gpt-5-mini` 在能力、稳定性与成本之间更平衡，适合“开发期高频验证”场景。
- 相比更大模型，`gpt-5-mini` 可显著降低日常测试成本，减少回归与调参开销。
- 相比更小模型，`gpt-5-mini` 能减少因能力不足导致的假失败，提升测试信号质量。

## 放弃了什么
- 方案 A：继续以 `gpt-4o-mini` 作为开发测试主模型。
  - 放弃原因：在当前阶段不再是最佳能力/成本平衡点。
- 方案 B：使用高阶大模型作为开发测试主模型。
  - 放弃原因：成本过高，不符合高频开发验证的经济性目标。
- 方案 C：以超低成本模型作为唯一主测试模型。
  - 放弃原因：复杂场景下稳定性不足，易引入误判。

## 影响
- M6 及后续阶段的开发测试计划、执行记录与报告应默认以 `gpt-5-mini` 为 OpenAI 主验证模型。
- 配置与脚本层应支持按场景切换模型，但报告中的“主结论”需基于 `gpt-5-mini` 结果。
- 若后续更换主测试模型，应新增决议或将本决议标记为 superseded。
